### 2.2 Shrinkage selection (held-out, not CV-5)

**Codex round-1 review (D-B-002c, D-B-008) flagged that the plan §7
wording "shrinkage CV" implied 5-fold inner cross-validation, while the
implementation in §2.7 uses a single held-out calibration split (the
same val partition used for early stopping). Two options were on the
table:**

- **Option A — true CV-5**: re-train the CNN five times per (dataset,
  seed) to get OOF nn predictions on the entire train set, then pick
  `s*`. Cost: 5× training. For r21 that is 39 datasets × 5 seeds × 5
  inner folds = 975 CNN training runs at ≈ 90 s each on the 4090 ≈ 24
  hours of GPU time, which crowds out FCK audit20 / full-57 work and
  pushes the multiseed result past the next checkpoint.
- **Option B — single held-out (chosen)**: reuse the val partition
  produced by `_make_loaders` (deterministic from `(seed, val_fraction)`)
  as the calibration split for `s*`. Zero extra training cost; the
  selection is unbiased w.r.t. the test set because the val partition is
  disjoint from the test by construction (random 80 / 20 split of the
  pre-defined train set). It is **biased w.r.t. the train set** (the
  CNN's parameters were tuned to minimise val loss), but the selection
  target is `argmin_s RMSE(y_val, z_val + s × nn_residual_val)`, not
  RMSE on train; the only thing the CNN's training "saw" of the val
  partition was the gradient-free early-stopping signal.

**Decision (this revision, queued as D-B-002c-revised
DECISION_PENDING_CODEX_REVIEW):** adopt Option B explicitly for r21.
Rationale: do-no-harm fallback dominates the design — `s = 0` is in the
grid, so the worst case is "back to teacher". The variance of the `s*`
selection on a 20 %-of-train held-out split is small (5 candidates,
unimodal RMSE-vs-s curve in practice), and the catastrophic-loss
diagnostic catches any (dataset, seed) where the held-out selection
fails. If r21 reveals high `s*` variance across seeds, Option A becomes
worth its 24 h budget; we can revisit after the multiseed result.

Per (dataset, seed):

1. Fit teacher with **OOF predictions on the full train split** via
   5-fold inner CV (the existing `_build_pls_residual_teacher_oof`
   already does this).
2. Train V2L on residuals = `y_train − teacher_oof_train`. Internal
   train / val split via `_make_loaders` (val_fraction = 0.2, seeded).
3. After CNN training, run inference on `X_val_s` to get
   `nn_val_residual = y_proc.inverse_transform(nn_val_scaled)`.
4. For each `s ∈ {0, 0.25, 0.5, 0.75, 1.0}`, compute
   `RMSE(y_val_orig, z_val + s × nn_val_residual)` where
   `z_val = z_train_oof[val_idx]`. Pick `s*` minimising this.
5. Final test prediction = `teacher_test + s* × nn_test`. Record `s*`
   and the per-s inner-CV RMSE in `hyperparams_json`.

`s = 0` is the do-no-harm fallback (test prediction = pure teacher).
Catastrophic-loss diagnostic: when the final test rmse exceeds the
teacher-only test rmse by more than `shrinkage_catastrophic_threshold`
(default +50 %), set `catastrophic = true` on the row. Post-run
aggregator filters to those rows for diagnosis.

### 2.7 Shrinkage CV — implementation pointers

Existing plumbing in `bench/nicon_v2/benchmarks/run_baseline_benchmark.py`
already provides the OOF teacher predictions (`z_train_raw` from
`_build_pls_residual_teacher_oof`, line ~978) and trains the CNN on
`y_train - z_train_raw` (line ~1216 onwards). The R20 prediction line
(~1276) is:

```python
pred = pred + z_test_raw          # implicit shrinkage = 1.0
```

For r21 the change is localised:

1. **Reuse training-time validation partition.** `_make_loaders` already
   returns `X_val` and `y_val`. Capture `val_idx` from the same RNG split
   so `z_val = z_train_raw[val_idx]` aligns. The split is reproducible
   (seeded), so this is a single-line addition (return `val_idx` from
   `_make_loaders` and propagate it up to `_run_torch_cnn`).
2. **Run CNN inference on val** after training:
   ```python
   nn_val_residual = y_proc.inverse_transform(
       predict_torch_regressor(model, X_val_s, device=device)
   )
   ```
   `X_val_s` is the same scaled X_val used by the val_loader during
   training; it is already in scope for the early-stopping logic.
3. **Pick `s*` over the candidate grid:**
   ```python
   import numpy as np
   candidates = (0.0, 0.25, 0.5, 0.75, 1.0)
   z_val = z_train_raw[val_idx]
   y_val_orig = y_proc.inverse_transform(y_val_scaled)
   rmses = [np.sqrt(np.mean((y_val_orig - (z_val + s * nn_val_residual)) ** 2))
            for s in candidates]
   s_star = candidates[int(np.argmin(rmses))]
   ```
4. **Apply at test:** replace the R20 line with
   `pred = z_test_raw + s_star * pred`. Record `s_star` and the
   per-`s` inner-CV RMSE in `hyperparams_json` for diagnostics.
5. **Catastrophic-loss flag** in `_result_row`:
   ```python
   teacher_test_rmse = rmse(y_test, z_test_raw)
   row["catastrophic"] = bool(rmsep_value > 1.5 * teacher_test_rmse)
   ```
   Threshold `1.5×` is the spec from §2.1. Tune per Codex review.

Cost: zero extra training (val partition is already computed for
early stopping). One additional `predict_torch_regressor` pass on
~20 % of train (≤ 50 ms on the 4090), one short candidate scan on
the host. Multi-seed orchestration (seeds 0-4) reuses the existing
runner's seed loop.

A new `Variant` is needed for r21:

```python
Variant("V2L-Residual-AOMPLS-shrinkage", family="nicon_v2a",
        extra={..., "pls_residual_teacher": "aompls_extended",
               "pls_residual_oof": True, "pls_oof_n_folds": 5,
               "shrinkage_cv": True,
               "shrinkage_grid": (0.0, 0.25, 0.5, 0.75, 1.0)}),
```

…wired through the `extra` parser at lines 1127–1135. Total estimated
diff: ~70 LOC across `run_baseline_benchmark.py` and `training.py`,
plus 1–2 unit tests in `bench/nicon_v2/tests/`.
