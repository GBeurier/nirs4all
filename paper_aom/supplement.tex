\section{Purpose of the Supplement}

This supplement documents the evidence chain behind the main manuscript.  It
is deliberately more operational than the article: local result sources are
listed, claims are mapped to files, deployable and oracle quantities are
separated, and reviewer-sensitive weaknesses are made explicit.  The goal is
to make the AOM study auditable rather than to hide ongoing work behind a
single compressed results table.

The supplement uses existing local artifacts only.  No additional benchmark was
launched to prepare this draft.

For a Talanta submission, the supplement should ultimately become the place
where full dataset tables, paired statistical tests, operator-selection
frequencies, failure cases, leakage audits and software reproducibility
metadata are made explicit.  The present version already establishes that
structure and marks the remaining experiments required before submission.

\section{Notation and Linear-Operator Scope}

Let $\X\in\mathbb{R}^{n\times p}$ be a centered spectral matrix and
$\Y\in\mathbb{R}^{n\times q}$ a centered response matrix.  A strict linear
spectral operator $\A_b\in\mathbb{R}^{p\times p}$ acts as
\begin{equation}
  \X_b = \X\A_b\transpose .
\end{equation}
The operator bank always contains the identity.  This makes standard PLS or
standard Ridge available to the selection procedure.

The strict-linear scope includes smoothing filters, derivative filters,
finite differences and detrending projections.  It does not include
sample-adaptive or fitted transformations such as SNV, MSC, EMSC or ASLS.
Those transformations are useful in NIRS but must be treated as fold-local
pipeline branches.  This distinction is central to the paper because the fast
AOM algebra is valid only for fixed operators inside a calibration fold.

\section{AOM-PLS Derivation}

\subsection{Covariance identity}

For a fixed linear operator $\A_b$, the transformed cross-covariance is
\begin{equation}
  \X_b\transpose \Y =
  (\X\A_b\transpose)\transpose\Y =
  \A_b\X\transpose\Y .
  \label{eq:sup_cov_identity}
\end{equation}
This identity permits candidate operators to be evaluated in covariance space
for SIMPLS and in adjoint form for NIPALS.  It avoids materializing every
transformed matrix $\X\A_b\transpose$ when the operator can be applied directly
to vectors or matrices.

\subsection{Original-space coefficients}

If a PLS component is estimated in transformed space with direction $r_a$, the
corresponding original-space direction is
\begin{equation}
  z_a = \A_b\transpose r_a .
\end{equation}
Collecting $k$ such directions gives $\Z=[z_1,\ldots,z_k]$ and scores
$\mathbf T=\X\Z$.  The final coefficient matrix is
\begin{equation}
  \B = \Z(\mathbf P\transpose\Z)^+\mathbf Q\transpose .
\end{equation}
Thus AOM-PLS remains a single linear calibration model on the original
wavelength grid.

\section{AOM-Ridge Derivation}

For centered $\X_c$ and $\Y_c$, Ridge regression can be solved in the dual.  A
single operator gives
\begin{align}
  \mathbf Z_b &= \X_c\A_b\transpose,\\
  \K_b &= \mathbf Z_b\mathbf Z_b\transpose
       = \X_c\A_b\transpose\A_b\X_c\transpose,\\
  \mathbf C &= (\K_b + \alpha \mathbf I_n)^{-1}\Y_c.
\end{align}
The original-space coefficient matrix is
\begin{equation}
  \bm\beta_b = \A_b\transpose\A_b\X_c\transpose\mathbf C.
\end{equation}

For a superblock or weighted operator mixture,
\begin{equation}
  \K = \sum_{b=1}^{m}s_b^2\X_c\A_b\transpose\A_b\X_c\transpose,
  \qquad
  \mathbf M = \sum_{b=1}^{m}s_b^2\A_b\transpose\A_b,
\end{equation}
and the original-space coefficient matrix is
\begin{equation}
  \bm\beta = \mathbf M\X_c\transpose(\K+\alpha\mathbf I_n)^{-1}\Y_c.
\end{equation}

\section{Algorithmic Sketches}

\subsection{Global AOM-PLS}

\begin{enumerate}
  \item Build a compact operator bank containing identity and a small set of
  smoothing, derivative and detrending operators.
  \item For each candidate number of components and each operator, evaluate
  calibration-fold performance.
  \item Select the operator and component count with the best calibration-fold
  criterion.
  \item Refit on the complete calibration set.
  \item Store selected operator, component count, original-space coefficients,
  centering constants and fold-level diagnostics.
\end{enumerate}

\subsection{AOM-Ridge}

\begin{enumerate}
  \item Build operator-induced kernels
  $\K_b=\X_c\A_b\transpose\A_b\X_c\transpose$ inside the calibration fold.
  \item Select $\alpha$ and operator geometry by nested calibration-fold
  validation.
  \item Solve the dual Ridge system.
  \item Recover original-space coefficients.
  \item Refit on the complete calibration set and evaluate once on the external
  test set.
\end{enumerate}

\section{Reproducibility Checklist}

\begin{enumerate}
  \item Record repository commit hash, Python version and package versions.
  \item Store the cohort manifest used for every table.
  \item Store split indices for every dataset.
  \item Store fold predictions and final test predictions.
  \item Store selected operators, component counts, $\alpha$ values and branch
  preprocessing choices.
  \item Store runtime and hardware metadata.
  \item Regenerate tables from scripts rather than manually editing manuscript
  tables.
\end{enumerate}

\section{Recommended Positioning}

The safest wording for the current draft is: AOM is a general linear-operator
framework; AOM-PLS has the strongest deployable full-cohort evidence; AOM-Ridge
is a mathematically consistent and promising extension with deployable evidence
and a stronger oracle envelope, but requires full multi-seed promotion before
being described as a production default.  This position is scientifically
stronger than claiming universal benchmark dominance.
